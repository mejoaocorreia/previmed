# Command: review-command-permission

## Objetivo

Rever se um comando pode ser executado.

## Processo

1. Identificar comando.
2. Identificar ambiente.
3. Identificar impacto.
4. Verificar dados/credenciais.
5. Verificar rollback.
6. Classificar risco.
7. Aprovar, bloquear ou pedir autorização.

## Regra

Comando destrutivo ou produção exige autorização explícita.
